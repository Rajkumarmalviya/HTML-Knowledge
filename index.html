What is HTML?

Definition

HTML (HyperText Markup Language) is the standard markup language for documents designed to be displayed in a web browser. It defines the structure and content of web pages.

Key Point: HTML is not a programming language. It's a markup language used to structure content on webpages.

Important Notes

  • HTML describes the structure of a web page semantically
  • HTML elements are the building blocks of HTML pages
  • HTML is not case-sensitive, but lowercase is the standard
  • HTML5 is the latest version with new semantic elements and APIs

HTML = "Hyper Text Markup Language"

HTML is not a Programming Language instead it's a Markup Language. It structures content on webpages.

Interactive Example

<!DOCTYPE html>
<html>
<head>
    <title>My First Page</title>
</head>
<body>

<h1>Welcome to HTML!</h1>
<p>This is a paragraph.</p>
<p>This is another paragraph.</p>

</body>
</html>

This is how the HTML code renders in a browser.

HTML Evolution Timeline

1991 - HTML 1.0

HTML 1.0 - The Beginning

The first version with basic elements like:

<h1> to <h6> <p> <a> <ul>, <li> <pre>

Simple structure for academic documents and research papers.

1995 - HTML 2.0

HTML 2.0 - Interactive Pages

Added support for forms, making interactive web pages possible.

<form> <input> <select> <textarea>

Enabled user input, search forms, and basic interactivity.

1997 - HTML 3.2 & 4.0

HTML 3.2 & 4.0 - Presentational HTML

Introduced presentational elements like <font> and <center>. Table-based layouts became popular.

<font> <center> <table>

This era saw mixing of structure with presentation, leading to messy code.

2000 - XHTML & CSS

XHTML & CSS - Separation of Concerns

CSS was introduced to separate styling from HTML content. XHTML enforced stricter syntax rules.

CSS (1996 onward) was introduced to:

  • Move styling out of HTML
  • Restore HTML's original purpose (structure)

XHTML (2000) was HTML rewritten as strict XML. Very unforgiving and failed in practice (too rigid for real-world web).

2014 - HTML5

HTML5 - The Modern Standard

The modern standard with semantic elements, native multimedia, and powerful APIs.

<header> <nav> <section> <article> <footer> <video> <audio> <canvas>

HTML5 introduced semantic tags, native audio/video, canvas for graphics, better forms, and APIs for web apps.

HTML Document Structure

A Simple HTML Document 
<!DOCTYPE html>
<html>
<head>
<title>Page Title</title>
</head>
<body>

<h1>My First Heading</h1>
<p>My first paragraph.</p>

</body>
</html>

Document Structure Explained

  • The <!DOCTYPE html> declaration defines that this document is an HTML5 document
  • The <html> element is the root element of an HTML page
  • The <head> element contains meta information about the HTML page
  • The <title> element specifies a title for the HTML page (shown in browser's title bar)
  • The <body> element defines the document's body, containing all visible contents

HTML Element Types

What is an HTML Element?

An HTML element is a structured unit of content defined by a tag, its attributes, and its content. HTML elements are classified in multiple ways. A real understanding comes from knowing all major classifications, not just one.

1. By Structure: Paired vs Void Elements

Type Description Examples Paired (Container) Elements They have opening tag, closing tag, and content inside. Used when content must be wrapped or grouped. <div></div>, <p></p>, <section></section> Void (Self-Closing) Elements Do not have closing tags, do not contain content. Represent standalone resources or actions. <img>, <br>, <input>, <meta>

2. By Display Behavior: Block vs Inline

Block vs Inline Visualization

<!-- Block elements -->
<div style="background: lightblue;">Block Element 1</div>
<p style="background: lightgreen;">Block Element 2</p>

<!-- Inline elements -->
<span style="background: lightcoral;">Inline 1</span>
<a href="#" style="background: lightyellow;">Inline 2</a>
<strong style="background: lightpink;">Inline 3</strong>

Block elements start on a new line and take full width.
Inline elements stay in line with text.

Type Characteristics Examples Use Block-Level Elements Start on a new line, take full available width, can contain other block and inline elements <div>, <p>, <section>, <article> Used for layout and structure Inline Elements Do not start on a new line, only take as much width as needed, usually contain text or other inline elements <span>, <a>, <strong>, <em> Used for text-level meaning

3. By Meaning: Semantic vs Non-Semantic

Professional Tip

Always prefer semantic elements when possible. They improve accessibility, SEO, and code maintainability.

Semantic Elements Non-Semantic Elements When to Use Non-Semantic Benefits of Semantic <header>, <nav>, <main> <div>, <span> When no semantic element fits, pure grouping is needed Accessibility, SEO, Readability, Maintainability <article>, <section>, <footer> Overuse is a beginner mistake Professionals strongly prefer semantic elements

4. By Content Model (What They Can Contain)

HTML defines rules for what elements may contain. Common categories:

  • Metadata elements: <title>, <meta>, <link>
  • Flow content: Most visible elements (<div>, <p>, <section>)
  • Phrasing content: Text-level elements (<span>, <a>, <strong>)
  • Embedded content: <img>, <video>, <iframe>

This prevents invalid HTML.

5. By Function: Specialized Element Types

Function Examples Purpose Text Content Elements <p>, <h1>-<h6>, <blockquote> Display and structure text content Grouping Elements <div>, <section>, <article> Group related content together Interactive Elements <button>, <input>, <select> Enable user interaction Media Elements <img>, <video>, <audio> Embed multimedia content Custom & Web Components <my-component></my-component> Modern HTML allows custom elements powered by JavaScript

Semantic HTML

What are Semantic Elements?

Semantic HTML uses elements that clearly describe their meaning in a human- and machine-readable way. Instead of using generic <div> and <span> elements, semantic elements tell browsers, developers, and assistive technologies what type of content they contain.

Benefits of Semantic HTML

  • Accessibility: Screen readers can better interpret page structure
  • SEO: Search engines understand content better
  • Maintainability: Code is easier to read and understand
  • Future-proof: Browsers can apply default styles appropriately

Key Semantic Elements

semantic-layout.html 
<!DOCTYPE html>
<html>
<head>
    <title>Semantic HTML Example</title>
</head>
<body>
    <!-- Header section -->
    <header>
        <h1>Website Title</h1>
        <p>Website slogan</p>
    </header>

    <!-- Navigation -->
    <nav>
        <ul>
            <li><a href="#">Home</a></li>
            <li><a href="#">About</a></li>
            <li><a href="#">Contact</a></li>
        </ul>
    </nav>

    <!-- Main content -->
    <main>
        <article>
            <header>
                <h2>Article Title</h2>
                <p>Published on <time datetime="2023-10-15">October 15, 2023</time></p>
            </header>
            
            <section>
                <h3>Introduction</h3>
                <p>Article content goes here...</p>
            </section>
            
            <aside>
                <h4>Related Articles</h4>
                <ul>
                    <li><a href="#">Related article 1</a></li>
                </ul>
            </aside>
        </article>
    </main>

    <!-- Footer -->
    <footer>
        <p>© 2023 Company Name</p>
        <address>
            Contact: <a href="mailto:contact@example.com">contact@example.com</a>
        </address>
    </footer>
</body>
</html>
Semantic Element Purpose When to Use <header> Introductory content or navigation Page header, article header, section header <nav> Navigation links Main navigation, table of contents, pagination <main> Main content of the document Primary content (only one per page) <article> Self-contained composition Blog post, news article, forum post <section> Thematic grouping of content Chapters, tabbed content, numbered sections <aside> Content indirectly related to main content Sidebars, pull quotes, advertisements <footer> Footer for its nearest section Page footer, article footer, section footer

HTML Forms

What are HTML Forms?

HTML Forms are used to collect user input. They contain various types of form controls like text fields, checkboxes, radio buttons, submit buttons, and more. Form data is sent to a server for processing.

 DOM Event System

                    
                    

                        
 What is a DOM Event?

                        
An event is a browser signal that something has happened (click, keypress, submit, scroll, load, etc.). Event listeners are JavaScript functions that respond to these events.

Event Delegation (Professional Technique)

                    
                    

                        
 Why Use Event Delegation?

                        
Instead of adding event listeners to every child element, add one to a parent and use event bubbling to handle events efficiently.

                    

                    
                    

                        

                            event-delegation.js
                            
                        

                        

                            
// Instead of this (inefficient):
document.querySelectorAll('.item').forEach(item => {
    item.addEventListener('click', handleClick);
});

// Do this (event delegation):
document.querySelector('.list').addEventListener('click', (e) => {
    // Check if clicked element is an item
    if (e.target.classList.contains('item')) {
        handleClick(e);
    }
});

Detailed Explanation

                    
                    

                        

                            
Stage 1: Receiving HTML

                            

                                
The browser downloads HTML as plain text. This includes the document structure, content, and references to external resources (CSS, JavaScript, images).

                            

                        

                        
                        

                            
Stage 2: Parsing & Tokenization

                            

                                
The browser's HTML parser reads the text character by character, converting it into tokens (start tags, end tags, attribute names, attribute values, text content).

                                
Critical Path: Parsing stops when scripts are encountered unless they are marked with async or defer.

                            

                        

                        
                        

                            
Stage 3: Building the DOM Tree

                            

                                
Tokens are converted into nodes, which are organized into a tree structure called the Document Object Model (DOM). Each HTML element becomes a DOM node with properties, methods, and relationships.

                                
The DOM is a live representation that JavaScript can manipulate.

                            

                        

                        
                        

                            
Stage 4: CSS Processing & Render Tree

                            

                                
While building the DOM, the browser also downloads and processes CSS to create the CSS Object Model (CSSOM). The DOM and CSSOM are combined to form the Render Tree.

                                
The Render Tree only contains visible elements (excludes display: none elements).

                            

                        

                        
                        

                            
Stage 5: Layout, Paint, and Composite

                            

                                
Layout (Reflow): Calculates the exact position and size of each element.

                                
Paint: Fills in pixels for each element (colors, borders, shadows).

                                
Composite: Layers are combined in correct order for display.

                            

                        

                    

                


                
                

                    
 HTML Multimedia

                    
                    

                        
 What is Multimedia in HTML?

                        
HTML Multimedia refers to integrating various media types like images, audio, video, and vector graphics into web pages. HTML5 introduced native support for audio and video without requiring plugins like Flash.

                    

                    
                    
Images

             image-medium.jpg 800w,
             image-large.jpg 1200w"
     sizes="(max-width: 600px) 480px,
            (max-width: 1000px) 800px,
            1200px"
     alt="Responsive image">

    <img src="chart.png" alt="Sales chart for 2023">
