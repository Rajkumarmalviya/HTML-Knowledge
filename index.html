Semantic HTML

What are Semantic Elements?

Semantic HTML uses elements that clearly describe their meaning in a human- and machine-readable way. Instead of using generic <div> and <span> elements, semantic elements tell browsers, developers, and assistive technologies what type of content they contain.

Benefits of Semantic HTML

  • Accessibility: Screen readers can better interpret page structure
  • SEO: Search engines understand content better
  • Maintainability: Code is easier to read and understand
  • Future-proof: Browsers can apply default styles appropriately

Key Semantic Elements

semantic-layout.html 
<!DOCTYPE html>
<html>
<head>
    <title>Semantic HTML Example</title>
</head>
<body>
    <!-- Header section -->
    <header>
        <h1>Website Title</h1>
        <p>Website slogan</p>
    </header>

    <!-- Navigation -->
    <nav>
        <ul>
            <li><a href="#">Home</a></li>
            <li><a href="#">About</a></li>
            <li><a href="#">Contact</a></li>
        </ul>
    </nav>

    <!-- Main content -->
    <main>
        <article>
            <header>
                <h2>Article Title</h2>
                <p>Published on <time datetime="2023-10-15">October 15, 2023</time></p>
            </header>
            
            <section>
                <h3>Introduction</h3>
                <p>Article content goes here...</p>
            </section>
            
            <aside>
                <h4>Related Articles</h4>
                <ul>
                    <li><a href="#">Related article 1</a></li>
                </ul>
            </aside>
        </article>
    </main>

    <!-- Footer -->
    <footer>
        <p>© 2023 Company Name</p>
        <address>
            Contact: <a href="mailto:contact@example.com">contact@example.com</a>
        </address>
    </footer>
</body>
</html>
Semantic Element Purpose When to Use <header> Introductory content or navigation Page header, article header, section header <nav> Navigation links Main navigation, table of contents, pagination <main> Main content of the document Primary content (only one per page) <article> Self-contained composition Blog post, news article, forum post <section> Thematic grouping of content Chapters, tabbed content, numbered sections <aside> Content indirectly related to main content Sidebars, pull quotes, advertisements <footer> Footer for its nearest section Page footer, article footer, section footer <figure> & <figcaption> Self-contained content with caption Images, diagrams, code snippets with descriptions <time> Machine-readable date/time Publication dates, event dates <mark> Highlighted text for reference Search results, important excerpts

Semantic vs Non-Semantic Comparison

Non-Semantic (Bad Practice)
<div id="header">
  <div class="title">My Site</div>
</div>

<div id="nav">
  <div class="menu">...</div>
</div>

<div class="main-content">
  <div class="post">
    <div class="post-title">...</div>
  </div>
</div>

<div id="footer">
  <div class="copyright">...</div>
</div>
Semantic (Good Practice)
<header>
  <h1>My Site</h1>
</header>

<nav>
  <ul>...</ul>
</nav>

<main>
  <article>
    <h2>...</h2>
  </article>
</main>

<footer>
  <p>...</p>
</footer>

The semantic version clearly communicates structure to browsers, screen readers, and search engines.

HTML Forms

What are HTML Forms?

HTML Forms are used to collect user input. They contain various types of form controls like text fields, checkboxes, radio buttons, submit buttons, and more. Form data is sent to a server for processing.

Form Structure & Elements

contact-form.html 
<form action="/submit-form" method="POST">
    
    <!-- Text input -->
    <div class="form-group">
        <label for="name">Full Name:</label>
        <input type="text" id="name" name="name" required placeholder="Enter your name">
    </div>

    <!-- Email input -->
    <div class="form-group">
        <label for="email">Email:</label>
        <input type="email" id="email" name="email" required>
    </div>

    <!-- Password -->
    <div class="form-group">
        <label for="password">Password:</label>
        <input type="password" id="password" name="password" minlength="8">
    </div>

    <!-- Dropdown -->
    <div class="form-group">
        <label for="country">Country:</label>
        <select id="country" name="country">
            <option value="us">United States</option>
            <option value="uk">United Kingdom</option>
            <option value="ca">Canada</option>
        </select>
    </div>

    <!-- Radio buttons -->
    <fieldset>
        <legend>Gender:</legend>
        <input type="radio" id="male" name="gender" value="male">
        <label for="male">Male</label><br>
        
        <input type="radio" id="female" name="gender" value="female">
        <label for="female">Female</label>
    </fieldset>

    <!-- Textarea -->
    <div class="form-group">
        <label for="message">Message:</label>
        <textarea id="message" name="message" rows="4"></textarea>
    </div>

    <!-- Submit button -->
    <button type="submit">Submit</button>
    <button type="reset">Reset</button>

</form>

 HTML Multimedia

                    
                    

                        
 What is Multimedia in HTML?

                        
HTML Multimedia refers to integrating various media types like images, audio, video, and vector graphics into web pages. HTML5 introduced native support for audio and video without requiring plugins like Flash.

                    

                    
                    
Images

                    
                    

                        

                            image-examples.html
                            
                        

                        

                            
<!-- Basic image -->
<img src="photo.jpg" alt="Description of photo">

<!-- Responsive image with srcset -->
<img src="image-small.jpg"
     srcset="image-small.jpg 480w,
             image-medium.jpg 800w,
             image-large.jpg 1200w"
     sizes="(max-width: 600px) 480px,
            (max-width: 1000px) 800px,
            1200px"
     alt="Responsive image">

<!-- Figure with caption -->
<figure>
    <img src="chart.png" alt="Sales chart for 2023">
    <figcaption>Figure 1: Quarterly sales data for 2023</figcaption>
</figure>

<!-- Picture element for art direction -->
<picture>
    <source media="(min-width: 1200px)" srcset="large.jpg">
    <source media="(min-width: 800px)" srcset="medium.jpg">
    <img src="small.jpg" alt="Responsive image">
</picture>

Web Components & Custom Elements

                    
                    

                        

                            custom-element.js
                            
                        

                        

                            
// Define a custom element
class UserCard extends HTMLElement {
    constructor() {
        super();
        
        // Create shadow DOM
        const shadow = this.attachShadow({mode: 'open'});
        
        // Create template
        const template = document.createElement('template');
        template.innerHTML = `
            <style>
                .user-card {
                    border: 1px solid #ccc;
                    border-radius: 8px;
                    padding: 20px;
                    max-width: 300px;
                    font-family: Arial, sans-serif;
                }
                .name {
                    font-size: 1.5rem;
                    font-weight: bold;
                    color: #333;
                }
                .email {
                    color: #666;
                    margin: 10px 0;
                }
            </style>
            <div class="user-card">
                <div class="name"><slot name="name">Unknown</slot></div>
                <div class="email"><slot name="email">No email</slot></div>
            </div>
        `;
        
        shadow.appendChild(template.content.cloneNode(true));
    }
}

// Register the custom element
customElements.define('user-card', UserCard);

Data Attributes & Microdata

                    
                    

                        
 Data Attributes

                        
Custom data attributes (data-*) allow you to store extra information on HTML elements that can be accessed via JavaScript.

                    

                    
                    

                        

                            data-attributes.html
                            
                        

                        

                            
<!-- Storing data in HTML elements -->
<div id="product"
     data-product-id="12345"
     data-price="29.99"
     data-category="electronics">
    Product Name
</div>

<!-- Accessing data in JavaScript -->
<script>
    const product = document.getElementById('product');
    const productId = product.dataset.productId; // "12345"
    const price = product.dataset.price; // "29.99"
    const category = product.dataset.category; // "electronics"
</script>

 How Browser Processes HTML

                    
                    

                        
 The Journey from HTML to Screen

                        
When a browser receives HTML, it goes through several stages to convert text into a visual webpage. Understanding this process helps optimize page performance and debug rendering issues.

Detailed Explanation

                    
                    

                        

                            
Stage 1: Receiving HTML

                            

                                
The browser downloads HTML as plain text. This includes the document structure, content, and references to external resources (CSS, JavaScript, images).

                            

                        

                        
                        

                            
Stage 2: Parsing & Tokenization

                            

                                
The browser's HTML parser reads the text character by character, converting it into tokens (start tags, end tags, attribute names, attribute values, text content).

                                
Critical Path: Parsing stops when scripts are encountered unless they are marked with async or defer.

                            

                        

                        
                        

                            
Stage 3: Building the DOM Tree

                            

                                
Tokens are converted into nodes, which are organized into a tree structure called the Document Object Model (DOM). Each HTML element becomes a DOM node with properties, methods, and relationships.

                                
The DOM is a live representation that JavaScript can manipulate.

                            

                        

                        
                        

                            
Stage 4: CSS Processing & Render Tree

                            

                                
While building the DOM, the browser also downloads and processes CSS to create the CSS Object Model (CSSOM). The DOM and CSSOM are combined to form the Render Tree.

                                
The Render Tree only contains visible elements (excludes display: none elements).

                            

                        

                        
                        

                            
Stage 5: Layout, Paint, and Composite

                            

                                
Layout (Reflow): Calculates the exact position and size of each element.

                                
Paint: Fills in pixels for each element (colors, borders, shadows).

                                
Composite: Layers are combined in correct order for display.

                            

                        

                    

                    
                    

                        
 Performance Implications

                        
    
                            
  • Minimize DOM size: Fewer elements = faster rendering
    • 
                            
  • CSS at the top: Load CSS early to avoid repaints
    • 
                            
  • JavaScript at the bottom: Or use async/defer
    • 
                            
  • Avoid layout thrashing: Multiple reads/writes to DOM cause reflows
    • 
                            
  • Use efficient selectors: CSS selector performance matters
    •
