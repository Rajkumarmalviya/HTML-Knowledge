What is HTML?

Definition

HTML (HyperText Markup Language) is the standard markup language for documents designed to be displayed in a web browser. It defines the structure and content of web pages.

Key Point: HTML is not a programming language. It's a markup language used to structure content on webpages.

Important Notes

  • HTML describes the structure of a web page semantically
  • HTML elements are the building blocks of HTML pages
  • HTML is not case-sensitive, but lowercase is the standard
  • HTML5 is the latest version with new semantic elements and APIs

Interactive Example

<!DOCTYPE html>
<html>
<head>
    <title>My First Page</title>

HTML Evolution Timeline

1991

HTML 1.0

The first version with basic elements like headings, paragraphs, links, and lists.

<h1> to <h6> <p> <a> <ul>, <li> <pre>
1995

HTML 2.0

Added support for forms, making interactive web pages possible.

<form> <input> <select> <textarea>
1997

HTML 3.2 & 4.0

Introduced presentational elements like <font> and <center>. Table-based layouts became popular.

<font> <center> <table>
2000

XHTML & CSS

CSS was introduced to separate styling from HTML content. XHTML enforced stricter syntax rules.

2014

HTML5

The modern standard with semantic elements, native multimedia, and powerful APIs.

<header> <nav> <section> <article> <footer> <video> <audio> <canvas>

HTML Document Structure

basic-html-document.html 
<!DOCTYPE html> declaration defines HTML5 document
<!DOCTYPE html>
<html lang="en">
<head>
    <!-- Meta information about the document -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
</head>
<body>
    <!-- All visible content goes here -->
    <h1>My First Heading</h1>
    <p>My first paragraph.</p>
</body>
</html>
Element Purpose Notes
<!DOCTYPE html> Defines document type and version Must be the first line in HTML5 documents
<html> Root element of the page Contains all other elements
<head> Contains metadata and links Not visible to users (except title)
<title> Page title shown in browser tab Important for SEO and bookmarking
<body> Contains all visible content Only one <body> per document

HTML Element Types

Remember

HTML elements can be classified in multiple ways. A complete understanding requires knowing all these classifications.

1. By Structure: Paired vs Void Elements

Type Description Examples Paired (Container) Elements Have opening and closing tags, contain content <div></div>, <p></p>, <section></section> Void (Self-Closing) Elements No closing tag, no content, represent standalone resources <img>, <br>, <input>, <meta>

2. By Display: Block vs Inline Elements

Block vs Inline Visualization

<!-- Block elements -->
<div style="background: lightblue;">Block Element 1</div>
<p style="background: lightgreen;">Block Element 2</p>

<!-- Inline elements -->
<span style="background: lightcoral;">Inline 1</span>
<a href="#" style="background: lightyellow;">Inline 2</a>
<strong style="background: lightpink;">Inline 3</strong>

Block elements start on a new line and take full width.
Inline elements stay in line with text.

3. By Meaning: Semantic vs Non-Semantic

Professional Tip

Always prefer semantic elements when possible. They improve accessibility, SEO, and code maintainability.

Semantic Elements Non-Semantic Elements When to Use Non-Semantic <header>, <nav> <div>, <span> When no semantic element fits <main>, <article> For pure styling/grouping <footer>, <section> Overuse is a beginner mistake

DOM (Document Object Model)

The Big Idea

HTML is text → DOM is objects in memory → Elements become nodes

When the browser parses HTML, every element becomes a DOM node with properties, methods, and relationships.

How HTML Maps to DOM

html-to-dom.html 
<!-- HTML Source -->
<!DOCTYPE html>
<html>
<head>
    <title>Demo</title>
</head>
<body>
    <h1>Hello</h1>
    <p>Welcome <strong>user</strong></p>
</body>
</html>

DOM Tree Structure

Document
 ├── DocumentType (html)
 ├── html (Element)
     ├── head (Element)
     │   └── title (Element)
     │       └── "Demo" (Text)
     └── body (Element)
         ├── h1 (Element)
         │   └── "Hello" (Text)
         └── p (Element)
             ├── "Welcome " (Text)
             └── strong (Element)
                 └── "user" (Text)

Every box is a DOM node. Text becomes separate text nodes.

Types of DOM Nodes

Node Type Created From Example
Document Node The document itself document
Element Nodes HTML elements HTMLParagraphElement, HTMLDivElement
Text Nodes Text inside elements "Hello" inside <p>
Attribute Nodes Element attributes src, alt on <img>

Why This Knowledge Matters

Understanding DOM helps you:

  • Avoid layout bugs
  • Understand React/Vue warnings
  • Debug rendering issues
  • Write efficient UI code

DOM Event System

What is a DOM Event?

An event is a browser signal that something has happened (click, keypress, scroll, etc.). Event listeners are JavaScript functions that respond to these events.

Event Delegation (Professional Technique)

Why Use Event Delegation?

Instead of adding event listeners to every child element, add one to a parent and use event bubbling to handle events efficiently.

event-delegation.js 
// Instead of this (inefficient):
document.querySelectorAll('.item').forEach(item => {
    item.addEventListener('click', handleClick);
});

// Do this (event delegation):
document.querySelector('.list').addEventListener('click', (e) => {
    // Check if clicked element is an item
    if (e.target.classList.contains('item')) {
        handleClick(e);
    }
});
Event Property Description Example event.target The element that triggered the event The button that was clicked event.currentTarget The element with the event listener The parent with the click handler event.stopPropagation() Stops event from bubbling up Prevents parent handlers from firing event.preventDefault() Prevents default browser action Stops form submission or link navigation

HTML Quick Reference

Essential HTML tags and their purposes:

Tag Use Example <!DOCTYPE html> Defines document as HTML5 First line of every HTML5 document <html> Root element of the page Contains all other elements <head> Container for metadata Title, styles, scripts, SEO info <body> Container for visible content Headings, paragraphs, images, links <h1> to <h6> Headings (h1 is most important) Page titles, section headings <p> Paragraph Blocks of text content <a> Hyperlink <a href="url">Link</a> <img> Image <img src="img.jpg" alt="text"> <ul>, <ol>, <li> Lists (unordered, ordered, list item) Navigation menus, item lists <div> Division/container Generic container for styling/layout <span> Inline container Style part of text <table>, <tr>, <td> Table, table row, table data Tabular data display <form>, <input> Form and input elements User input, search, login forms <header> Header section Top of page, usually contains logo/nav <nav> Navigation section Main navigation links <main> Main content area Primary content of the page <footer> Footer section Bottom of page, copyright, links

Quick Tips

Always include <!DOCTYPE html> as the first line.

Use semantic elements for better accessibility.

Close all tags properly to avoid rendering issues.
